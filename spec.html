Blitzcrank

A Go gateway that connects Seerr webhooks and media-server automations to a Pi agent

1. Overview

Blitzcrank is now a narrow Go gateway for Seerr/Jellyfin media-server operations. It receives Seerr issue webhooks, runs scheduled Markdown automations, owns service credentials and gateway state, and delegates all agentic reasoning to Pi through Pi RPC mode.

Pi owns the model/provider configuration, skills, durable agent sessions, tool loop, and conversation history. Blitzcrank owns webhook ingress, SQLite issue state, Seerr final comments/resolution, service API credentials, and a small internal HTTP tool gateway.

Current scope: Seerr issue workflows and scheduled automations only. Discord support, the old native LLM runtime, Codex/OpenAI/OpenRouter clients, Deno TypeScript sandboxing, and old generic agent thread storage have been removed.

2. Features

Seerr Issue Gateway

Accepts Seerr webhooks, deduplicates events, locks per issue, runs Pi, posts final comments, and resolves issues when Pi returns RESOLVE_ISSUE: yes.

Pi Agent Runtime

Blitzcrank starts pi --mode rpc with persistent sessions, project skills, and a project extension that registers media-server tools.

Typed Service Tools

Pi calls seerr_request, jellyfin_request, sonarr_request, radarr_request, and sabnzbd_request; Go injects credentials and enforces service-relative paths.

Markdown Automations

Scheduled Markdown tasks run through Pi with one durable Pi session per automation.

Gateway Persistence

SQLite stores only Seerr issue gateway state. Pi JSONL sessions store agent conversation history.

Trace Search

JSONL traces remain searchable through thread_history_search for repeated issues and automation history.

4. Workflows

4.1 Seerr Issue Workflow

  1. Seerr sends an issue webhook to seerr.webhook_path.
  2. Blitzcrank validates the optional webhook secret, classifies the event, and ignores unknown/duplicate/bot-authored events.
  3. The harness locks on the Seerr issue id and loads/updates the SQLite issue thread.
  4. Blitzcrank builds a compact issue prompt from the latest payload, prior events, prior runs, and trace compactions.
  5. Pi runs in the persistent session for issue:<id>, loads .pi/skills/seerr-issue-solver/SKILL.md, and calls typed tools.
  6. Pi returns RESOLVE_ISSUE: yes/no followed by the public Seerr comment.
  7. Blitzcrank validates, signs, posts the final comment, and resolves the issue only for RESOLVE_ISSUE: yes.

4.2 Automation Workflow

Automations are Markdown files under automations/*.md with front matter: name, description, and schedule. Currently @hourly is supported. Each automation runs through Pi with a stable session named from automation:<name>.

The current bundled automation, hourly-stale-import-handler, uses Sonarr/Radarr queue and manual-import APIs to import safe stale downloads or remove clearly rejected stale queue items. It reports only concrete new imports, removals, or new manual-review blockers.

5. Tool System

Pi sees a small typed tool surface registered by .pi/extensions/blitzcrank-tools.ts. The extension calls Blitzcrank's internal HTTP gateway, and Go performs the service request with configured credentials.

All service request tools require purpose. Paths must be service-relative and must not contain full URLs or credentials. Non-GET requests require safety_level = "narrow_mutation" and safety_reason.

6. Pi Runtime

Blitzcrank uses Pi as the only agent runtime. Provider setup, API keys, OAuth, default models, skill loading, context management, compaction, and tool-loop behavior belong to Pi.

Blitzcrank controls only which Pi binary to call, working directory, persistent session location, per-task model override, and tool gateway connection.

[pi]
command = "pi"
cwd = "."
sessions_dir = "threads/pi-sessions"
tool_base_url = "http://127.0.0.1:8080"
tool_secret = "change-me"

[pi.models]
default = "anthropic/claude-sonnet-4-5:high"
seerr = ""
automation = ""

Thinking/reasoning is configured inline in Pi's model string, for example anthropic/claude-sonnet-4-5:high.

8. Persistence & State

SQLite

SQLite stores only Blitzcrank gateway state:

  • issue_threads — Seerr issue lifecycle and status.
  • issue_thread_events — webhook event records and dedupe keys.
  • issue_runs — Pi run metadata, posted status, attribution, errors, and completion reason.

Old generic agent/Discord tables were removed. Pi owns agent conversation history.

Pi sessions

Pi sessions are stored under pi.sessions_dir, or runtime.threads_dir/pi-sessions when unset. Blitzcrank creates one durable session per Seerr issue and per automation using sanitized thread ids such as issue-42.jsonl and automation-hourly-stale-import-handler.jsonl.

JSONL traces

  • issues/issue-<id>.jsonl — webhook events, tool calls, progress comments, runs, and completion.
  • issues/issue-<id>.compactions.jsonl — issue prompt compaction records.

These traces power thread_history_search.

9. Safety Model

  • Small tool surface: Pi only sees service request tools and trace search.
  • Credential isolation: API keys remain in Go config and are injected by the gateway, not placed in Pi prompts or tool arguments.
  • Relative paths only: service tools reject full URLs and credential-looking path parameters.
  • Mutation friction: non-GET requests require narrow_mutation and a concrete safety reason.
  • Harness-owned Seerr lifecycle: Pi cannot post final comments or resolve issues directly.
  • Evidence before action: skills require read-only service checks and validation after mutations.
  • Privacy: final comments must avoid raw JSON, internal URLs, service credentials, unrelated user data, and implementation details.

10. Deployment Notes

Run locally:

cp config.example.toml blitzcrank.toml
go run ./cmd/blitzcrank

Useful checks:

go test ./...
go build ./cmd/blitzcrank
nix build

Provider credentials are configured through Pi's normal environment/settings, for example ANTHROPIC_API_KEY, OPENAI_API_KEY, or Pi auth storage. Blitzcrank should not own provider setup.

Keep PI_TOOL_SECRET secret. It protects the internal tool gateway used by Pi extensions.
